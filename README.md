# Parallel Example: Quest Seed Usability Lab

This packet models how a game-development interest could become a research journal and paper-lite draft. It is adjacent to game design and generative AI work, but it does not describe the student's actual project.

## Core idea

The imagined Space is a **Quest Seed Usability Lab**. A user gives a short game concept and asks several small language models to generate quest seeds. The project does not ask whether the output is "good" in a vague way. It asks whether the output would actually help a game developer make something.

## Research question

Do realistic project-based tests show more about AI tool quality than generic benchmark-style claims do?

## Space idea

Inputs:

- Game genre: fantasy, sci-fi, horror, cozy, or mystery
- Quest type: rescue, exploration, puzzle, boss setup, or moral choice
- Project constraint: target player, tone, setting, and one must-include object

Outputs:

- Three generated quest seeds from different models or prompt styles
- A developer-usefulness checklist
- Notes on what would need rewriting before use in an actual game

## What this packet demonstrates

- How to turn broad generative-AI curiosity into a practical evaluation question
- How to treat curation and testing as research evidence
- How to connect a small Space to procedural content generation research
- How to write honestly when the project is early-stage

## How to use this example

Read `sources.md` first to see the research area, then `research-journal.md` to see how the question develops. `PAPER.md` shows a finished paper-lite shape that could be revised with real outputs.

Borrow the structure, not the claims. A real version should include actual prompts, outputs, and a saved comparison table.
